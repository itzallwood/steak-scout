# CLAUDE.md вҖ” Steak Scout Project

## Project Goal
Build an AI-powered agent that scrapes local grocery stores, meat markets, and butchers
to find the best deals on steaks. This is a learning project focused on understanding
web scraping fundamentals, AI agents, and LLM integration.

---

- Python-comfortable developer, new to web scraping and AI agents
- Using Claude Code in VS Code integrated with GitHub
- Prioritize clear, readable, well-commented code over clever one-liners

## Target Stores вҖ” Confirmed Intelligence

All stores below have been researched and the scraping approach verified.

### Phase 1 вҖ” Local Butchers (Static HTML, requests + BeautifulSoup)

| Store | URL | Platform | Notes |
|---|---|---|---|
| America's Choice Gourmet | `americaschoicegourmet.com/product-category/beef/` | WordPress/WooCommerce | Two-phase scrape: category page вҶ’ product detail page for weight. Weight in description text, extract with regex. Prices per item, not per lb. |
| The Butcher's Market (Wilmington) | `wilmington.thebutchersmarkets.com/Meat-and-Seafood-Beef-Steaks/` | Custom e-commerce | Static HTML, clean URL structure, no login required. Straightforward category page scrape. |

**WooCommerce scraper pattern (Americas Choice Gourmet):**
- Phase 1: Scrape `/product-category/beef/` for name, price, sale_price, product URL
- Phase 2: Visit each product URL, extract short description
- Use regex to pull weight: `re.search(r'(\d+\.?\d*)\s*(oz|lb)', description, re.IGNORECASE)`
- Default `weight_value` and `weight_unit` to `None` if regex finds nothing вҖ” don't crash

**Standard output dict for ALL scrapers:**
```python
{
  "store": "America's Choice Gourmet",
  "cut": "Filet Mignon",
  "price": 27.00,
  "sale_price": None,       # original price if on sale, else None
  "weight_value": 8.0,
  "weight_unit": "oz",
  "price_unit": "per_item", # or "per_lb"
  "url": "https://...",
  "scraped_at": "2026-03-26T14:00:00"
}
```

### Phase 2 вҖ” Walmart (requests + JSON extraction)

| Store | URL | Approach | Difficulty |
|---|---|---|---|
| Walmart | `walmart.com/search?q=steak+beef` | requests + JSON parse | вӯҗвӯҗ Medium |

**Key discovery:** Walmart embeds ALL product data in a `<script id="__NEXT_DATA__">` tag
as a JSON object вҖ” no JavaScript rendering needed. Use `requests` + `BeautifulSoup` to
grab the script tag, then `json.loads()` to parse it.

**Confirmed data path:**
```python
data = json.loads(soup.find("script", {"id": "__NEXT_DATA__"}).text)
items = data["props"]["pageProps"]["initialData"]["searchResult"]["itemStacks"][0]["items"]

# Each item has:
item["name"]                        # "Beef Choice Angus NY Strip Steak, 0.54-1.56 lb"
item["priceInfo"]["linePriceDisplay"]  # "$15.52" (tray price)
item["priceInfo"]["unitPrice"]         # "$20.97/lb"
item["priceInfo"]["finalCostByWeight"] # True/False вҖ” priced per lb or flat
```

**Main challenge:** Bot detection. Walmart blocks plain requests quickly.
Mitigations to implement in order:
1. Rotate realistic User-Agent headers
2. Add `time.sleep(2-5)` between requests
3. Add `Referer` and `Accept-Language` headers to mimic a real browser
4. If still blocked, use Playwright as fallback (it worked in browser testing)

**Location note:** Walmart search results are location-aware based on IP.
Running locally will naturally return your nearest Wilmington store's inventory.

### Phase 3 вҖ” Lowe's Foods (Playwright + cookie injection)

| Store | URL | Approach | Difficulty |
|---|---|---|---|
| Lowe's Foods | `shop.lowesfoods.com` | Playwright | вӯҗвӯҗвӯҗ Medium-Hard |

**Key finding:** `shop.lowesfoods.com` returns completely empty HTML to a plain
`requests` call вҖ” 100% JavaScript-rendered React app. Playwright required.

**Location strategy:** Prices are the same online as in-store (no location markup).
Set store via Playwright by clicking the store picker UI once, then extract and
hardcode the resulting store cookie for all future scrape runs.

### Phase 4 вҖ” Hard Targets (Playwright + possible proxy)

| Store | Difficulty | Notes |
|---|---|---|
| Costco | вӯҗвӯҗвӯҗ Hard | JS-rendered, login preferred for full pricing |
| Kroger / Harris Teeter | вӯҗвӯҗвӯҗвӯҗ Very Hard | JS-rendered, location-gated, strong bot detection |
| Whole Foods | вӯҗвӯҗвӯҗвӯҗ Very Hard | React app, Amazon infrastructure, aggressive bot detection |

For these, consider a scraping API service (ScrapingBee, Oxylabs) if direct
Playwright scraping gets blocked consistently.

## Architecture Overview

```
User runs agent
      в”Ӯ
      в–ј
 price_agent.py  (LangGraph agent)
      в”Ӯ
      в”ңв”Җв”Җ Tool: scrape_store(store_name, url)
      в”Ӯ         в””в”Җв”Җ Calls the right scraper class
      в”Ӯ                  в”ңв”Җв”Җ requests + BeautifulSoup  (simple sites)
      в”Ӯ                  в””в”Җв”Җ Playwright                (JS-heavy sites)
      в”Ӯ
      в”ңв”Җв”Җ Tool: save_prices(store, cut, price)
      в”Ӯ         в””в”Җв”Җ Writes to SQLite via db/queries.py
      в”Ӯ
      в”ңв”Җв”Җ Tool: get_best_deals(cut)
      в”Ӯ         в””в”Җв”Җ Queries SQLite, returns ranked results
      в”Ӯ
      в””в”Җв”Җ Tool: generate_report()
                в””в”Җв”Җ report/generator.py вҶ’ prices_report.html
```

The LLM (Claude) acts as the brain of the agent вҖ” it decides which tools to call,
in what order, and how to interpret results. LangGraph manages the state and flow
between tool calls.

## Key Conventions

### Scraper Design
- Every scraper inherits from `BaseScraper` in `base_scraper.py`
- Each scraper must implement a `scrape()` method that returns a list of dicts:
  ```python
  [{"store": "Kroger", "cut": "ribeye", "price_per_lb": 12.99, "url": "..."}]
  ```
- Never hardcode URLs in scrapers вҖ” pass them in or load from config
- Always include a `time.sleep()` between requests to be a polite scraper
- Log what you're doing at each step using Python's `logging` module

### Database
- Single SQLite file at `data/prices.db`
- Always store a `scraped_at` timestamp with every price record
- Never delete old records вҖ” this builds a price history over time

### Agent Tools
- Each LangGraph tool should do one thing only (single responsibility)
- Tools return structured dicts, not raw strings
- The agent should explain its reasoning in plain English before each tool call

### Environment Variables
Required in `.env` (see `.env.example`):
```
ANTHROPIC_API_KEY=your_key_here
```

### HTML Report
- Single self-contained `.html` file (no external dependencies)
- Generated fresh each run, saved to project root as `prices_report.html`
- Should show: best deal per cut, price comparison table, when data was last scraped
- Aim for a clean, readable design вҖ” this is a functional report, not a dashboard app

## What NOT to Do
- Don't scrape aggressively вҖ” always add delays between requests
- Don't commit `.env` or `data/prices.db` to GitHub
- Don't build the agent before the scrapers work вҖ” get data flowing first
- Don't use async everywhere from the start вҖ” introduce it only where Playwright requires it
- Don't over-engineer the database schema early вҖ” keep it simple and expand later
